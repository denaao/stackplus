5 findings críticos

Bloqueadores para produção madura. Atacar primeiro.

SEC-001

Webhook ANNAPAY sem HMAC

Comparação direta de string no lugar de verificação HMAC-SHA256. Qualquer um com o secret consegue forjar webhooks e injetar transações PIX falsas.

stackplus-api/src/modules/banking/annapay.controller.ts:124-131
▸ Fix: Implementar createHmac('sha256', secret).update(rawBody).digest('hex') e comparar com X-Signature header.
SEC-006

Webhook sem idempotency (race condition)

Check-then-update vulnerável: dois webhooks simultâneos podem registrar a mesma transação duas vezes antes do UPDATE consolidar.

stackplus-api/src/modules/banking/annapay.service.ts:1175-1176
▸ Fix: INSERT ... ON CONFLICT DO NOTHING com unique constraint em chargeId, ou transação com lock pessimista.
DATA-001

Campo de auditoria sem foreign key

registeredBy é String solta em Transaction e PrepaidChargePending. Sem @relation, permite referência órfã e compromete a trilha de auditoria financeira.

stackplus-api/prisma/schema.prisma:538, :561
▸ Fix: Migration registeredByUserId → @relation(fields: [registeredByUserId], references: [id]) com índice e backfill.
QUAL-001

Cobertura de testes ~zero

Backend: 3 arquivos de teste (auth, homegame, session — todos mockados). Frontend: zero. Vitest configurado mas subutilizado.

Backend: 3 testes
Frontend: 0 testes
▸ Fix: Prioridade: annapay.service.ts, cashout flow, reconciliação bancária e cálculo de acerto. Meta: 70% nas camadas críticas.
QUAL-002

Zero pipeline de CI/CD

Nenhum .github/workflows/. Deploy manual (Vercel + Railway no push). Nada valida TS/lint/testes antes de mergar.

▸ Fix: Workflow mínimo em PR: npm run lint && tsc --noEmit && npm run build. Adicionar testes quando existirem.

Challenge phase — o que pode estar errado

Auto-crítica do orquestrador antes de fechar.

Cobertura da auditoria

3 agents varreram em paralelo — security, data, quality. Cada finding tem evidência arquivo:linha. Suficiente pra panorama, insuficiente pra compliance PCI/SOC2.

RLS do Postgres não testado

Analisei só o schema Prisma. Se tem policies no PG elas podem estar desatualizadas ou ausentes em tabelas novas.

Runtime não exercitado

Não rodei o backend local — pode ter memory leak em socket.io, race conditions em cashier/session não visíveis no código.

Frontend raso

Não analisei XSS, CSRF, armazenamento de JWT (provável localStorage — revisar). Escopo ficou backend-heavy.

Final verdict

O projeto tem arquitetura sólida e entregou valor (LP no ar, app em produção, pagamentos funcionando). Mas está em zona cinzenta pra escala: a integração ANNAPAY é o calcanhar de Aquiles — sem HMAC e sem idempotency, um atacante com acesso ao secret consegue injetar transações. Zero testes significa que qualquer refactor pode quebrar algo silenciosamente.

Prioridade absoluta: os 5 HIGH do Sprint 0. Os outros 16 são melhorias graduais que podem entrar no backlog normal, atacadas em ordem de criticidade.
